Completing the Instruction Set

Implementing the remaining opcodes in sequential order

The Final Push

After implementing the core instructions needed to get Tetris running (Part 1 and Part 2), this iteration completes the CPU instruction set by systematically adding all remaining opcodes. Rather than implementing only what's needed by specific games, this approach ensures comprehensive coverage of the Game Boy's instruction set.

This implementation focused on going from top to bottom through the missing opcodes, including all CB-prefixed instructions that weren't needed for basic gameplay but are essential for full compatibility.

What's New in This Branch

This implementation adds the following instructions that weren't in previous iterations:

  • Accumulator Rotates: RLCA, RRCA, RLA, RRA (faster 4-cycle versions)
  • Arithmetic with Carry: ADC A, r/(HL)/n8 - all 9 variants
  • Subtraction: SUB r/(HL)/n8 and SBC A, r/(HL)/n8 - 18 variants total
  • Special Operations: DAA (BCD), SCF, CCF (carry flag), DEC (HL)
  • Control: HALT, STOP (power-saving modes)
  • CB-Prefixed Instructions: All 256 bit manipulation operations (RLC, RRC, RL, RR, SLA, SRA, SRL, BIT, SET - RES was already implemented)

Note: Instructions like ADD, XOR, AND, OR, CP, LD variants, PUSH/POP, jumps, calls, and 16-bit operations were already implemented in previous iterations.

Rotate and Shift Instructions (Accumulator)

The Game Boy has four special rotate instructions that operate only on the accumulator (register A). These are faster (4 cycles) than their CB-prefixed counterparts (8 cycles) and are commonly used for quick bit manipulations.

RLCA - Rotate Left Circular Accumulator (0x07)

Rotates A left. Bit 7 is copied to both bit 0 and the carry flag.

int CPU::rlca() {
    bool msb = (af.high & 0x80) != 0;
    af.high = (af.high << 1) | (msb ? 1 : 0);

    // Set flags
    set_flag(af.low, FLAG_ZERO, false);
    set_flag(af.low, FLAG_SUBTRACT, false);
    set_flag(af.low, FLAG_HALF_CARRY, false);
    set_flag(af.low, FLAG_CARRY, msb);

    pc += 1;
    return 4;
}

Example: If A = 0b10110101, after RLCA: A = 0b01101011, Carry = 1

RRCA - Rotate Right Circular Accumulator (0x0F)

Rotates A right. Bit 0 is copied to both bit 7 and the carry flag.

int CPU::rrca() {
    bool lsb = (af.high & 0x01) != 0;
    af.high = (af.high >> 1) | (lsb ? 0x80 : 0);

    // Set flags
    set_flag(af.low, FLAG_ZERO, false);
    set_flag(af.low, FLAG_SUBTRACT, false);
    set_flag(af.low, FLAG_HALF_CARRY, false);
    set_flag(af.low, FLAG_CARRY, lsb);

    pc += 1;
    return 4;
}

RLA - Rotate Left Through Accumulator (0x17)

Rotates A left through carry. Bit 7 goes to carry, carry goes to bit 0.

int CPU::rla() {
    bool msb = (af.high & 0x80) != 0;
    af.high = (af.high << 1) | (get_flag(af.low, FLAG_CARRY) ? 1 : 0);

    // Set flags
    set_flag(af.low, FLAG_ZERO, false);
    set_flag(af.low, FLAG_SUBTRACT, false);
    set_flag(af.low, FLAG_HALF_CARRY, false);
    set_flag(af.low, FLAG_CARRY, msb);

    pc += 1;
    return 4;
}

RRA - Rotate Right Through Accumulator (0x1F)

Rotates A right through carry. Bit 0 goes to carry, carry goes to bit 7.

int CPU::rra() {
    bool lsb = (af.high & 0x01) != 0;
    af.high = (af.high >> 1) | (get_flag(af.low, FLAG_CARRY) ? 0x80 : 0);

    // Set flags
    set_flag(af.low, FLAG_ZERO, false);
    set_flag(af.low, FLAG_SUBTRACT, false);
    set_flag(af.low, FLAG_HALF_CARRY, false);
    set_flag(af.low, FLAG_CARRY, lsb);

    pc += 1;
    return 4;
}

Note: Unlike their CB-prefixed counterparts, all four accumulator rotates (RLCA, RRCA, RLA, RRA) always clear the zero flag regardless of the result. This distinguishes them from the CB-prefixed versions which do set Z based on the result.

DAA - Decimal Adjust Accumulator (0x27)

DAA is one of the most complex Game Boy instructions. It adjusts the accumulator after BCD (Binary Coded Decimal) arithmetic to ensure the result represents a valid BCD number.

Understanding BCD

In BCD, each byte represents two decimal digits (0-99). For example:

  • 0x42 represents decimal 42
  • 0x99 represents decimal 99

When you add BCD numbers using normal binary addition, you sometimes get invalid results. For example, 0x09 + 0x01 = 0x0A, but in BCD this should be 0x10 (decimal 10). DAA corrects these results.

Implementation

int CPU::daa() {
    uint8_t a = af.high;
    uint8_t correction = 0;
    
    // Check if we need to adjust the lower nibble
    if (get_flag(af.low, FLAG_HALF_CARRY) || 
        (!get_flag(af.low, FLAG_SUBTRACT) && (a & 0x0F) > 9)) {
        correction |= 0x06;
    }
    
    // Check if we need to adjust the upper nibble
    if (get_flag(af.low, FLAG_CARRY) || 
        (!get_flag(af.low, FLAG_SUBTRACT) && a > 0x99)) {
        correction |= 0x60;
        set_flag(af.low, FLAG_CARRY, true);
    }
    
    // Apply correction (add or subtract based on last operation)
    if (get_flag(af.low, FLAG_SUBTRACT)) {
        a -= correction;
    } else {
        a += correction;
    }
    
    af.high = a;
    set_flag(af.low, FLAG_ZERO, a == 0);
    set_flag(af.low, FLAG_HALF_CARRY, false);
    
    pc += 1;
    return 4;
}

How DAA Works

DAA examines the accumulator and flags to determine what correction is needed:

  1. If the lower nibble is > 9 or half-carry is set, add/subtract 0x06 to/from the lower nibble
  2. If the upper nibble is > 9 or carry is set, add/subtract 0x60 to/from the upper nibble
  3. The direction (add or subtract) depends on the subtract flag from the previous operation

Example:

; Add 29 + 1 in BCD
LD A, $29    ; A = 0x29 (BCD for 29)
ADD A, $01   ; A = 0x2A (invalid BCD), H flag set
DAA          ; A = 0x30 (BCD for 30), corrected!

Carry Flag Manipulation

SCF - Set Carry Flag (0x37)

Sets the carry flag to 1, clears N and H flags.

int CPU::scf() {
    set_flag(af.low, FLAG_CARRY, true);
    set_flag(af.low, FLAG_SUBTRACT, false);
    set_flag(af.low, FLAG_HALF_CARRY, false);
    pc += 1;
    return 4;
}

CCF - Complement Carry Flag (0x3F)

Inverts the carry flag, clears N and H flags.

int CPU::ccf() {
    bool carry = get_flag(af.low, FLAG_CARRY);
    set_flag(af.low, FLAG_CARRY, !carry);
    set_flag(af.low, FLAG_SUBTRACT, false);
    set_flag(af.low, FLAG_HALF_CARRY, false);
    pc += 1;
    return 4;
}

These instructions are useful for multi-byte arithmetic and bit manipulations where you need precise control over the carry flag.

8-bit Arithmetic - Complete Families

ADD A, r/n8 (0x80-0x87, 0xC6)

Adds a value to A and stores the result in A.

int CPU::add_a(uint8_t value, int length, int cycles) {
    uint16_t result = static_cast<uint16_t>(af.high) + static_cast<uint16_t>(value);

    set_flag(af.low, FLAG_ZERO, (result & 0xFF) == 0);
    set_flag(af.low, FLAG_SUBTRACT, false);
    set_flag(af.low, FLAG_HALF_CARRY, ((af.high & 0x0F) + (value & 0x0F)) > 0x0F);
    set_flag(af.low, FLAG_CARRY, result > 0xFF);

    af.high = static_cast<uint8_t>(result & 0xFF);
    pc += length;
    return cycles;
}

Variants: ADD A, B/C/D/E/H/L/A/(HL)/n8

ADC A, r/n8 (0x88-0x8F, 0xCE)

Adds a value plus the carry flag to A. Essential for multi-byte addition.

int CPU::adc_a(uint8_t value, int length, int cycles) {
    uint16_t carry = get_flag(af.low, FLAG_CARRY) ? 1 : 0;
    uint16_t result = static_cast<uint16_t>(af.high) + 
                      static_cast<uint16_t>(value) + carry;

    set_flag(af.low, FLAG_ZERO, (result & 0xFF) == 0);
    set_flag(af.low, FLAG_SUBTRACT, false);
    set_flag(af.low, FLAG_HALF_CARRY, 
             ((af.high & 0x0F) + (value & 0x0F) + carry) > 0x0F);
    set_flag(af.low, FLAG_CARRY, result > 0xFF);

    af.high = static_cast<uint8_t>(result & 0xFF);
    pc += length;
    return cycles;
}

SUB r/n8 (0x90-0x97, 0xD6)

Subtracts a value from A.

int CPU::sub_a(uint8_t value, int length, int cycles) {
    uint16_t result = static_cast<uint16_t>(af.high) - static_cast<uint16_t>(value);

    set_flag(af.low, FLAG_ZERO, (result & 0xFF) == 0);
    set_flag(af.low, FLAG_SUBTRACT, true);
    set_flag(af.low, FLAG_HALF_CARRY, (af.high & 0x0F) < (value & 0x0F));
    set_flag(af.low, FLAG_CARRY, af.high < value);

    af.high = static_cast<uint8_t>(result & 0xFF);
    pc += length;
    return cycles;
}

SBC A, r/n8 (0x98-0x9F, 0xDE)

Subtracts a value and the carry flag from A. Essential for multi-byte subtraction.

int CPU::sbc_a(uint8_t value, int length, int cycles) {
    uint16_t carry = get_flag(af.low, FLAG_CARRY) ? 1 : 0;
    uint16_t result = static_cast<uint16_t>(af.high) - 
                      static_cast<uint16_t>(value) - carry;

    set_flag(af.low, FLAG_ZERO, (result & 0xFF) == 0);
    set_flag(af.low, FLAG_SUBTRACT, true);
    set_flag(af.low, FLAG_HALF_CARRY, (af.high & 0x0F) < ((value & 0x0F) + carry));
    set_flag(af.low, FLAG_CARRY, af.high < (value + carry));

    af.high = static_cast<uint8_t>(result & 0xFF);
    pc += length;
    return cycles;
}

Each family includes variants for all 8-bit registers and (HL) indirect access.

DEC (HL) - Decrement Memory (0x35)

While INC (HL) was already implemented, DEC (HL) was missing. This instruction decrements the byte pointed to by HL.

int CPU::dec_mem_hl(Memory& memory) {
    uint8_t value = memory.read(hl.pair);
    uint8_t result = value - 1;

    // Set flags
    set_flag(af.low, FLAG_ZERO, result == 0);
    set_flag(af.low, FLAG_SUBTRACT, true);
    set_flag(af.low, FLAG_HALF_CARRY, (result & 0x0F) == 0x0F);

    memory.write(hl.pair, result);
    pc += 1;
    return 12;
}

The half-carry flag is set when there's a borrow from bit 4. When decrementing from 0xX0 to 0xXF, the lower nibble goes from 0 to F, so the half-carry is set.

HALT and STOP

HALT (0x76)

HALT puts the CPU into a low-power state until an interrupt occurs. The implementation sets the halted flag, which the main execution loop checks:

int CPU::halt() {
    // NOTE: HALT bug not implemented
    halted = true; // Set the CPU to halted state
    pc += 1; // Move past the instruction
    return 4; // HALT takes 4 cycles
}

Main Loop Integration

The emulator's main loop checks the halted flag and handles wake conditions:

// Check for wake from HALT
uint8_t if_reg = memory.read(0xFF0F); // Interrupt Flag
uint8_t ie_reg = memory.read(0xFFFF); // Interrupt Enable
uint8_t pending = if_reg & ie_reg & 0x1F;

if(cpu.halted && pending) {
    cpu.halted = false; // Wake from HALT
}

if(cpu.halted || cpu.stopped) {
    cycles = 4; // Consume 4 cycles while halted/stopped
} else {
    handle_interrupts();
    cycles = step(); // Execute instruction
}

When halted, the CPU skips instruction execution but continues to consume cycles. Any enabled interrupt will wake the CPU from HALT.

STOP (0x10)

STOP is more drastic than HALT - it stops both CPU and PPU until a joypad interrupt occurs:

int CPU::stop() {
    stopped = true;
    pc += 2; // STOP is 2 bytes: 0x10 0x00
    return 4; // STOP takes 4 cycles
}

The main loop handles STOP wake conditions and prevents PPU updates:

// Check for wake from STOP (joypad interrupt only)
if(cpu.stopped && (if_reg & INT_JOYPAD)) {
    cpu.stopped = false; // Wake from STOP
}

// Skip instruction execution while stopped
if(cpu.halted || cpu.stopped) {
    cycles = 4;
} else {
    cycles = step();
}

// PPU doesn't update while stopped
if(!cpu.stopped) {
    ppu.step(cycles, memory);
}

The HALT Bug

The Game Boy has a hardware bug in HALT: if IME (Interrupt Master Enable) is disabled and there's a pending interrupt, the CPU doesn't halt. Instead, it continues execution but fails to increment PC after the HALT instruction, causing the next instruction to be executed twice.

Note: The HALT bug is not currently implemented in this emulator. This bug exists in all original Game Boy models, but most games don't encounter it because they typically enable interrupts before using HALT. Implementing it would require tracking a halt_bug flag and modifying the instruction fetch logic.

CGB Speed Switching

On Game Boy Color, STOP can be used to switch between normal and double-speed mode when the KEY1 register (0xFF4D) bit 0 is set. This functionality is not implemented in this emulator as it focuses on original Game Boy compatibility.

CB-Prefixed Instructions

The 0xCB prefix indicates a two-byte instruction for bit manipulation. These instructions provide comprehensive bit-level operations. Prior to this branch, only SWAP and RES were implemented. This branch adds all remaining CB operations.

Instruction Decoder

The CB instruction set has a regular structure that allows for efficient decoding:

int CPU::cb_execute_instruction(Memory& memory) {
    uint8_t opcode = memory.read(pc + 1);
    uint8_t reg_index = opcode & 0x07;  // Last 3 bits identify the register
    
    // Register lookup table (6 means (HL) indirect)
    uint8_t* regs[] = {&bc.high, &bc.low, &de.high, &de.low, 
                       &hl.high, &hl.low, nullptr, &af.high};
    
    // Dispatch based on opcode ranges...
}

The lower 3 bits of the opcode identify which register to operate on:

  • 0 = B, 1 = C, 2 = D, 3 = E, 4 = H, 5 = L, 6 = (HL), 7 = A

CB Rotate and Shift Instructions

RLC r/(HL) - Rotate Left Circular (0xCB 0x00-0x07)

Similar to RLCA but can operate on any register and sets the zero flag.

int CPU::rlc_r(uint8_t& reg) {
    bool msb = (reg & 0x80) != 0;
    reg = (reg << 1) | (msb ? 1 : 0);

    set_flag(af.low, FLAG_ZERO, reg == 0);
    set_flag(af.low, FLAG_SUBTRACT, false);
    set_flag(af.low, FLAG_HALF_CARRY, false);
    set_flag(af.low, FLAG_CARRY, msb);

    pc += 2;
    return 8;
}

RRC r/(HL) - Rotate Right Circular (0xCB 0x08-0x0F)

Rotates right, copying bit 0 to both bit 7 and carry.

int CPU::rrc_r(uint8_t& reg) {

    reg = (reg >> 1) | (lsb ? 0x80 : 0);

    set_flag(af.low, FLAG_HALF_CARRY, false);
    set_flag(af.low, FLAG_CARRY, lsb);

    pc += 2;
    return 8;
}

RL r/(HL) - Rotate Left Through Carry (0xCB 0x10-0x17)

Rotates left with bit 7 going to carry and carry going to bit 0.

int CPU::rl_r(uint8_t& reg) {
    bool msb = (reg & 0x80) != 0;

}

RR r/(HL) - Rotate Right Through Carry (0xCB 0x18-0x1F)

Rotates right with bit 0 going to carry and carry going to bit 7.

int CPU::rr_r(uint8_t& reg) {
    bool lsb = (reg & 0x01) != 0;
    reg = (reg >> 1) | (get_flag(af.low, FLAG_CARRY) ? 0x80 : 0);

    set_flag(af.low, FLAG_ZERO, reg == 0);
    set_flag(af.low, FLAG_SUBTRACT, false);
    set_flag(af.low, FLAG_HALF_CARRY, false);
    set_flag(af.low, FLAG_CARRY, lsb);

    pc += 2;

}

SLA r/(HL) - Shift Left Arithmetic (0xCB 0x20-0x27)

Shifts left, inserting 0 into bit 0. Bit 7 goes to carry.

int CPU::sla_r(uint8_t& reg) {
    bool msb = (reg & 0x80) != 0;
    reg <<= 1;
    reg &= 0xFE; // Ensure bit 0 is 0

    set_flag(af.low, FLAG_ZERO, reg == 0);
    set_flag(af.low, FLAG_SUBTRACT, false);
    set_flag(af.low, FLAG_HALF_CARRY, false);
    set_flag(af.low, FLAG_CARRY, msb);

    pc += 2;
    return 8;
}

SRA r/(HL) - Shift Right Arithmetic (0xCB 0x28-0x2F)

Shifts right, preserving bit 7 (sign bit). Bit 0 goes to carry.

int CPU::sra_r(uint8_t& reg) {
    bool msb = (reg & 0x80) != 0;
    bool lsb = (reg & 0x01) != 0;
    reg = (reg >> 1) | (msb ? 0x80 : 0); // Preserve sign bit

    set_flag(af.low, FLAG_ZERO, reg == 0);
    set_flag(af.low, FLAG_SUBTRACT, false);
    set_flag(af.low, FLAG_HALF_CARRY, false);
    set_flag(af.low, FLAG_CARRY, lsb);

    pc += 2;
    return 8;
}

SRL r/(HL) - Shift Right Logical (0xCB 0x38-0x3F)

Shifts right, inserting 0 into bit 7. Bit 0 goes to carry.

int CPU::srl_r(uint8_t& reg) {
    bool lsb = (reg & 0x01) != 0;
    reg >>= 1; // Bit 7 becomes 0

    set_flag(af.low, FLAG_ZERO, reg == 0);
    set_flag(af.low, FLAG_SUBTRACT, false);
    set_flag(af.low, FLAG_HALF_CARRY, false);
    set_flag(af.low, FLAG_CARRY, lsb);

    pc += 2;
    return 8;
}

SWAP r/(HL) (0xCB 0x30-0x37)

SWAP exchanges the upper and lower nibbles of a byte. This is unique to the Game Boy and is useful for BCD operations and sprite manipulation. (Already implemented in previous iterations, included here for completeness.)

int CPU::swap_r(uint8_t& reg) {
    reg = (reg << 4) | (reg >> 4);

    set_flag(af.low, FLAG_ZERO, reg == 0);
    set_flag(af.low, FLAG_SUBTRACT, false);
    set_flag(af.low, FLAG_HALF_CARRY, false);
    set_flag(af.low, FLAG_CARRY, false);

    pc += 2;
    return 8;
}

Example: If B = 0x3F, after SWAP B: B = 0xF3

SWAP is particularly useful for graphics routines where you need to manipulate nibbles representing pixel data or color palettes.

Bit Test, Reset, and Set

BIT b, r/(HL) (0xCB 0x40-0x7F)

Tests if a specific bit is set. Sets the zero flag if the bit is 0. This is a new addition in this branch.

int CPU::bit_r(uint8_t& reg, uint8_t bit) {
    set_flag(af.low, FLAG_ZERO, !(reg & (1 << bit)));
    set_flag(af.low, FLAG_SUBTRACT, false);
    set_flag(af.low, FLAG_HALF_CARRY, true);
    pc += 2;
    return 8;
}

The half-carry flag is always set to 1 by BIT, and the subtract flag is always cleared. The carry flag is unchanged.

RES b, r/(HL) (0xCB 0x80-0xBF)

Resets (clears) a specific bit to 0. No flags are affected. (This was already implemented in previous iterations.)

int CPU::res(uint8_t bit, uint8_t& reg) {
    reg &= ~(1 << bit);
    pc += 2;
    return 8;
}

SET b, r/(HL) (0xCB 0xC0-0xFF)

Sets a specific bit to 1. No flags are affected. This is a new addition in this branch.

int CPU::set(uint8_t bit, uint8_t& reg) {
    reg |= (1 << bit);
    pc += 2;
    return 8;
}

Decoding Bit Position

For BIT, RES, and SET instructions, the bit position (0-7) is encoded in the opcode:

// BIT instructions (0x40-0x7F): bit = (opcode - 0x40) / 8
// RES instructions (0x80-0xBF): bit = (opcode - 0x80) / 8
// SET instructions (0xC0-0xFF): bit = (opcode - 0xC0) / 8

if (opcode >= 0x40 && opcode <= 0x7F) {
    uint8_t bit = (opcode - 0x40) / 8;  // 0-7
    if (reg_index == 6) return bit_mem_hl(memory, bit);
    return bit_r(*regs[reg_index], bit);
}

This encoding allows for 8 bits × 8 registers = 64 opcodes per operation type.

Memory Variants of CB Instructions

All CB instructions except BIT have a (HL) variant that operates on memory. These take twice as long as register operations (16 vs 8 cycles) due to memory access.

Example - SWAP (HL):

int CPU::swap_mem_hl(Memory& memory) {
    uint8_t value = memory.read(hl.pair);
    value = (value << 4) | (value >> 4);

    set_flag(af.low, FLAG_ZERO, value == 0);
    set_flag(af.low, FLAG_SUBTRACT, false);
    set_flag(af.low, FLAG_HALF_CARRY, false);
    set_flag(af.low, FLAG_CARRY, false);

    memory.write(hl.pair, value);
    pc += 2;
    return 16; // Double cycles for memory access
}

The pattern is consistent across all CB instructions:

  • Read from (HL)
  • Perform operation
  • Set flags (if applicable)
  • Write back to (HL)
  • 16 cycles (except BIT, which is 12)
